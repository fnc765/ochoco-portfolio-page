# X（Twitter）おはつい自動取得の調査結果

## 結論

2026年3月時点で、X（Twitter）のツイートを**完全自動で取得する無料の手段は実質的に存在しません**。
半自動（共有ボタンから2タップ）での運用が現実的な最善策です。

## 調査した方法と結果

### 1. X（Twitter）公式 API

| 項目 | 内容 |
|------|------|
| 状態 | 有料化済み |
| 費用 | Basic プラン: 月額 $100〜 |
| 判定 | 個人利用には高すぎるため **不採用** |

2023年にAPI v1.1 が廃止され、無料枠（Free tier）ではツイートの読み取りができなくなりました。投稿のみ可能な無料枠はありますが、自分のツイートの取得には Basic プラン以上が必要です。

### 2. IFTTT（If This Then That）

| 項目 | 内容 |
|------|------|
| 状態 | X連携が機能停止 |
| 原因 | X側のAPI制限変更により、IFTTT の X トリガーが動作しなくなった |
| 判定 | **利用不可** |

以前は「自分がツイートしたら Webhook を呼ぶ」という自動化が無料でできましたが、現在は X のトリガーが正常に動作しません。

### 3. Zapier / Make (旧 Integromat)

| 項目 | 内容 |
|------|------|
| 状態 | X連携に制限あり |
| 費用 | 無料枠あり（制限付き） |
| 判定 | X のAPI制限により **実質利用不可** |

IFTTT と同様、X のAPI制限の影響を受けています。

### 4. スクレイピング

| 項目 | 内容 |
|------|------|
| 状態 | 技術的には可能だが非推奨 |
| リスク | 利用規約違反、アカウント凍結の可能性 |
| 判定 | リスクが高いため **不採用** |

Puppeteer 等でログインしてスクレイピングする方法はありますが、X の利用規約に違反する可能性が高く、アカウント凍結のリスクがあります。

### 5. RSS フィード

| 項目 | 内容 |
|------|------|
| 状態 | X が公式RSSを廃止済み |
| 代替 | Nitter 等のサードパーティが不安定 |
| 判定 | 安定性に欠けるため **不採用** |

### 6. FixTweet API（個別ツイート取得のみ）

| 項目 | 内容 |
|------|------|
| 状態 | 動作中（無料・APIキー不要） |
| 制限 | ツイートURLを知っている必要がある（自動巡回不可） |
| 判定 | **現在の collect API で採用済み** |

ツイートURLを指定すれば本文・画像・いいね数等を取得できます。ただし「新しいツイートを自動検知」する機能はないため、URLは人間が指定する必要があります。

## 採用した方式: 半自動収集

### 方式概要

```
[X アプリ] → 共有ボタン → [HTTP Request Shortcuts] → POST /api/collect → [D1 DB]
```

### メリット

- **無料**: 追加コストなし
- **確実**: 自分で選んだツイートだけを登録するため誤登録がない
- **簡単**: 共有メニューから2タップで完了
- **安定**: X のAPI制限に依存しない
- **即時**: 投稿直後に登録可能

### デメリット

- 完全自動ではない（毎日2タップの操作が必要）
- 登録を忘れる可能性がある

### 登録忘れ対策

- admin画面（`/admin/`）からの手動登録でいつでも追加可能
- 過去のツイートもURLさえあれば登録できる

## 将来的な可能性

| 条件 | 実現できること |
|------|----------------|
| X がAPI無料枠を復活させた場合 | IFTTT 等による完全自動化 |
| X が公式RSSを復活させた場合 | RSSリーダー経由の自動検知 |
| 安定したサードパーティAPIが出た場合 | cron による定期自動取得 |

現時点ではいずれも見通しが立たないため、半自動方式が最善です。
