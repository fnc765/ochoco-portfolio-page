## 📝 実装の詳細解説

### パーティクル数の選定理由

**50個を選択した理由**:
- ✅ 視覚的に十分な密度
- ✅ パフォーマンスへの影響が少ない
- ✅ bikyu83.comと同等の見た目
- ✅ モバイル端末でもスムーズに動作

### 移動速度の調整

**1.3倍速を選択した理由**:
- ✅ 遅すぎず、速すぎない自然な速度
- ✅ 背景として適切（注意を引きすぎない）
- ✅ インタラクションが楽しめる速度

### 透明度設定の工夫

**0.6 (60%) を基本値にした理由**:
- ✅ 背景として適切な控えめさ
- ✅ 接続線が見やすい
- ✅ コンテンツの邪魔にならない

**0.2〜0.6の範囲でアニメーション**:
- ✅ 微妙な明滅で動的な印象
- ✅ 完全に消えないため存在感を保つ

### 接続線の設定

**距離150px、透明度0.2を選択した理由**:
- ✅ 適度な距離で自然な繋がり
- ✅ 線が多すぎず、少なすぎない
- ✅ 透明度が低いため背景として機能

### pointer-events: noneの重要性

```css
pointer-events: none;
```

**この設定がないと**:
- ❌ ボタンがクリックできない
- ❌ リンクが機能しない
- ❌ テキスト選択ができない

**この設定により**:
- ✅ パーティクルはマウスイベントを透過
- ✅ 背後のコンテンツが正常に機能
- ✅ tsParticlesのインタラクションは`interactivity.detect_on: "canvas"`で別途実装

## ⚠️ 注意事項・制限事項

### CDN依存

**現在の実装**:
```html
<script src="https://cdn.jsdelivr.net/npm/tsparticles@2.12.0/tsparticles.bundle.min.js"></script>
```

**制限**:
- ❌ オフライン環境では動作しない
- ❌ CDNがダウンすると表示されない
- ❌ 中国など一部地域でCDNがブロックされる可能性

**解決策**:
1. ローカルにファイルをダウンロード
2. 別のCDN（unpkg等）をフォールバックとして追加
3. オフライン時の代替デザインを用意

### ブラウザ互換性

**動作確認済み**:
- ✅ Chrome（最新版）
- ✅ Firefox（最新版）
- ✅ Safari（最新版）
- ✅ Edge（最新版）

**未対応**:
- ❌ Internet Explorer 11以下

**理由**:

## 🔍 トラブルシューティング

### よくある問題と解決策

#### 問題1: パーティクルが表示されない

**原因A**: tsParticlesが読み込まれていない
**解決策**:
- ブラウザの開発者ツール（F12）でエラー確認
- CDNのURLが正しいか確認
- インターネット接続を確認

**原因B**: `#particles-js` 要素がない
**解決策**:
- `index.html` に `<div id="particles-js"></div>` があるか確認

**原因C**: `initParticles()` が呼ばれていない
**解決策**:
- `main.js` の `window.addEventListener('load')` 内に記述があるか確認

#### 問題2: ボタンがクリックできない

**原因**: `pointer-events: none` が設定されていない
**解決策**:
```css
#particles-js {
    pointer-events: none; /* この行を追加 */
}
```

#### 問題3: パーティクルが重すぎる（カクカクする）

**原因**: パーティクル数が多すぎる
**解決策**:
```javascript
number: {
    value: 30, // 50から30に減らす
}
```

#### 問題4: パーティクルが見えにくい

**原因**: 透明度が低すぎる
**解決策**:
```javascript
opacity: {
    value: 0.8, // 0.6から0.8に増やす
}
```

#### 問題5: 接続線が多すぎる/少なすぎる

**解決策**:
```javascript
line_linked: {
    distance: 200, // 150から調整（大きくすると線が増える）
}
```
