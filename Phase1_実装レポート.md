3. **@keyframes fadeIn**
   - opacity: 0 → 1
   - translateY: 20px → 0

### 3. js/main.js

**実装した機能**:

#### 3.1 ローディング画面制御
```javascript
window.addEventListener('load', () => {
  // 1秒間ローディング表示
  // → 0.5秒でフェードアウト
  // → ローダー非表示 + コンテンツ表示
});
```

**タイムライン**:
1. ページ読み込み完了
2. 1秒待機（ローディング表示）
3. 0.5秒でローダーをフェードアウト（opacity: 0）
4. ローダーを非表示（display: none）
5. コンテンツを表示（display: block）
6. コンテンツにフェードインアニメーション適用

#### 3.2 画像エラー処理
- プロフィール画像が見つからない場合
- 背景色を#333に設定
- alt属性を更新

#### 3.3 スムーズスクロール

- ページ内リンク（`href="#..."`）に対応

## 🖼️ 画像準備ガイド

### 必要な画像

#### 1. 背景画像 (`images/background.jpg`)
**推奨仕様**:
- 解像度: 1920×1080以上
- フォーマット: JPG, PNG, WebP
- サイズ: 500KB以下（圧縮推奨）
- 内容: 風景、抽象的パターン、グラデーションなど

**注意点**:
- ブラー効果がかかるため、細かい文字などは避ける
- 暗めの画像が推奨（テキストが白色のため）

#### 2. プロフィール画像 (`images/profile.jpg`)
**推奨仕様**:
- 解像度: 500×500px以上（正方形）
- フォーマット: JPG, PNG
- サイズ: 100KB以下
- 内容: 顔写真、アバター、ロゴなど

**注意点**:
- 円形にトリミングされるため、中央に被写体を配置
- 背景は無地または単純なものが推奨

### 画像の配置方法
1. `html+css+js\images\` フォルダを開く
2. 上記仕様に合った画像を準備
3. ファイル名を `background.jpg` と `profile.jpg` にリネーム
4. imagesフォルダに配置
5. ブラウザを更新して確認

## 📞 トラブルシューティング

### よくある問題と解決策

#### 問題1: ローディング画面が消えない
**原因**: JavaScriptが読み込まれていない
**解決策**:
- ブラウザの開発者ツール（F12）でエラー確認
- `js/main.js` のパスが正しいか確認

#### 問題2: スタイルが適用されない
**原因**: CSSファイルが読み込まれていない
**解決策**:
- `css/style.css` のパスが正しいか確認
- ブラウザのキャッシュをクリア（Ctrl+F5）

#### 問題3: フォントが表示されない
**原因**: Google Fontsが読み込めていない
**解決策**:
- インターネット接続を確認
- CDNリンクが正しいか確認

#### 問題4: 画像が表示されない
**原因**: 画像ファイルが配置されていない
**解決策**:
- `images/` フォルダに画像があるか確認
- ファイル名が `background.jpg`、`profile.jpg` か確認
- 大文字小文字を確認（WindowsとLinuxで挙動が異なる）
